## 安全与对齐

### OpenAI

#### Introducing the OpenAI Safety Fellowship
*发布：2026-04-06 · 类型：产品/公告 · 分类：safety*

**发布内容：** OpenAI 推出 Safety Fellowship 研究资助计划，旨在支持外部研究者开展 AI 安全、对齐与治理相关研究，并扩大安全研究社区规模。该项目定位为短期密集研究计划，面向具有技术或跨学科背景的研究者。

**项目结构：**
该 Fellowship 为期约五个月，计划时间为 **2026‑09‑14 至 2027‑02‑05**。入选研究者将在项目期间获得 OpenAI 提供的资金支持（stipend）、API credits 以及与 OpenAI 研究人员合作的机会。

研究主题包括：
- AI safety evaluation 与 red‑teaming
- robustness 与 adversarial testing
- agent oversight 与可控性
- privacy‑preserving safety
- misuse mitigation 与安全部署机制

**项目形式：**
研究者可以远程参与，并与 OpenAI 安全团队定期讨论研究方向。最终成果预计包括论文、benchmark、工具或数据集等公开研究产出。

**意义：**
随着 frontier models 能力快速增长，AI 安全研究正逐渐从单一模型评测扩展到 agent 系统安全与部署治理。OpenAI 通过 Fellowship 形式引入更多外部研究力量，试图建立更广泛的安全研究生态。

## Agent 开发

### Anthropic

#### Claude Sonnet 4.6
*发布：2026-04-07 · 类型：产品/模型 · 分类：agent-dev*

**发布内容：** Claude Sonnet 4.6 是 Anthropic Sonnet 系列的最新升级版本，在保持较低成本的同时显著提升了编码、复杂任务执行以及 computer‑use 能力，并首次引入 **1M token 上下文窗口（beta）**。

**核心能力：**
- 长上下文推理：支持约 100 万 token 输入，可在单次请求中处理大型代码库、合同文档或研究资料
- 更强 instruction following：在复杂任务中更少出现偏离指令或错误执行
- computer‑use 能力提升：在 OSWorld benchmark 中性能显著优于 предыдущий Sonnet 模型

**开发者体验：**
早期用户测试表明，开发者在 Claude Code 中约 **70% 情况下更偏好 Sonnet 4.6 而非 Sonnet 4.5**，并在部分任务中甚至优于 Opus 4.5。

**适用场景：**
- agent coding 与自动化开发
- 文档理解与企业知识处理
- 复杂多步骤任务规划

**意义：**
Sonnet 4.6 将接近 frontier 模型的能力下放到更低价格层级，使企业在构建 agent 系统时能够以更低成本获得高质量推理能力。

### Anthropic

#### Claude Opus 4.6
*发布：2026-04-07 · 类型：产品/模型 · 分类：agent-dev*

**发布内容：** Claude Opus 4.6 是 Anthropic 当前最强模型版本，重点强化 agentic coding、长上下文推理与复杂知识工作任务执行能力，同时引入 1M token 上下文窗口（beta）。

**核心能力：**
- 更强代码生成与代码审查能力
- 长时间 autonomous agent workflows
- 更稳定的多工具调用与任务规划

**评测结果：**
- Terminal‑Bench 2.0：agent coding 任务排名第一
- Humanity’s Last Exam：多学科推理能力领先
- GDPval‑AA：在金融与法律等知识工作任务中领先约 **144 Elo**

**企业应用能力：**
早期用户报告显示，该模型在大型代码库迁移、bug detection 以及复杂软件工程任务中表现接近高级工程师水平。

**意义：**
Opus 4.6 强化了长上下文 reasoning 与 agent planning 能力，是 Anthropic 推动“AI teammates”战略的重要一步。

## 模型预训练

### Google DeepMind

#### Gemma 4: Byte for byte, the most capable open models
*发布：2026-04-02 · 类型：研究/模型 · 分类：pretraining*

**核心贡献：** Google 发布 Gemma 4 开源模型系列，目标是在“单位参数性能（intelligence‑per‑parameter）”方面达到新的水平，使开发者能够在较小硬件上运行高性能模型。

**模型系列：**
Gemma 4 提供四种规模：
- **E2B**（Edge 2B）
- **E4B**（Edge 4B）
- **26B MoE**（Mixture‑of‑Experts）
- **31B Dense**

其中 31B 模型在 Arena AI open‑model leaderboard 上排名第三，26B MoE 排名第六，并在多个测试中击败规模大 20 倍以上的模型。

**技术特点：**
- 支持 **多模态输入**（图像、视频）
- 原生 **函数调用与 JSON 输出**，便于构建 agent
- 上下文窗口：128K（edge models）至 256K（大模型）
- 支持 **140+ 语言**

**部署目标：**
Gemma 4 被设计为可在多种硬件环境运行，包括：
- Android 设备
- 笔记本 GPU
- 单卡 H100
- 消费级 GPU

**开源策略：**
模型以 **Apache 2.0 许可证**发布，允许商业使用和二次开发。

**意义：**
Gemma 4 延续 Google 的 open‑model 战略，旨在构建“Gemini + Gemma”组合生态：Gemini 提供闭源 frontier 模型，Gemma 提供可本地部署的开源能力。

## 产品 / API 动态

### Google

#### New ways to balance cost and reliability in the Gemini API
*发布：2026-04-02 · 类型：产品 · 分类：product-api*

**发布内容：** Google 为 Gemini API 引入 **Flex 与 Priority 两种新的推理服务层级（inference tiers）**，帮助开发者在成本、延迟与可靠性之间进行更灵活的权衡。

**Flex Inference：**
- 成本约降低 **50%**
- 延迟与可靠性较低
- 适用于后台任务或 agent “thinking” 工作流

典型场景包括：
- 数据处理
- research simulations
- 大规模 agent 推理任务

**Priority Inference：**
- 提供最高可靠性
- 在平台高负载情况下优先处理
- 若超出额度可自动降级到 Standard tier

典型场景：
- 实时客服
- 内容审核
- 用户交互系统

**开发者接口：**
开发者可通过 `service_tier` 参数在 API 请求中指定服务层级。

**意义：**
随着 AI 应用从聊天工具扩展到复杂 agent 系统，推理需求呈现出明显的“后台计算 vs 实时交互”差异。Flex / Priority tiers 为开发者提供更精细的成本控制与可靠性管理机制。
