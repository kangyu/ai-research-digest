## 安全与对齐

### OpenAI

#### Introducing the OpenAI Safety Fellowship
*发布：2026-04-06 · 类型：产品/公告 · 分类：safety*

**发布内容：** OpenAI 启动 Safety Fellowship 计划，面向外部研究者资助 AI safety 与 alignment 相关研究。

**关键信息：**
- 项目周期：2026‑09‑14 → 2027‑02‑05
- 提供 stipend、API credits 与计算资源
- 研究方向：safety evaluation、robustness、agent oversight、privacy‑preserving safety、misuse mitigation 等

**适用对象：** 独立研究者、工程师或跨学科研究人员（CS、cybersecurity、HCI、social science 等）。

**目标：** 产出论文、benchmark、dataset 等可公开的研究成果，并推动安全研究社区发展。

## 补充文章

### Anthropic

#### Claude Sonnet 4.6
*发布：2026-04-07 · 类型：研究/模型 · 分类：agent-dev*

**核心贡献：** Claude Sonnet 4.6 在 Sonnet 系列中首次引入 1M token 长上下文，并显著提升代码生成、复杂任务规划与 computer‑use 能力。

**关键能力：**
- 更强 instruction following 与多步骤任务执行
- 改进 computer‑use 能力（OSWorld benchmark）
- 长上下文推理，可一次处理大型代码库或长文档

**意义：** 将接近 Opus 级别的推理能力下放到更低成本模型层级，适用于大规模 agent 与企业自动化场景。

### Google DeepMind

#### Gemma 4: Byte for byte, the most capable open models
*发布：2026-04-02 · 类型：研究/模型 · 分类：pretraining*

**核心贡献：** Google 发布 Gemma 4 开源模型系列，强调“单位参数性能（intelligence‑per‑parameter）”，面向本地部署与 agent workflow。

**模型结构：**
- E2B / E4B edge models
- 26B MoE
- 31B dense

**能力：**
- 多语言（140+）
- 原生多模态输入（图像 / 视频 / 音频）
- 长上下文（128K–256K）

**意义：** 为开发者提供高性能开源模型，可在从移动设备到 GPU 工作站的多种硬件环境运行。

### Google

#### New ways to balance cost and reliability in the Gemini API
*发布：2026-04-02 · 类型：产品 · 分类：product-api*

**发布内容：** Gemini API 新增 Flex 与 Priority 两种推理服务层级，用于在成本、延迟和可靠性之间进行权衡。

**Flex Inference：**
- 成本降低约 50%
- 适用于后台任务或 latency‑tolerant 工作流

**Priority Inference：**
- 提供最高可靠性
- 在高负载情况下保证关键请求

**意义：** 为 agent 与生产系统提供更灵活的推理成本控制机制。

## Agent 开发

### Anthropic

#### Claude Opus 4.6
*发布：2026-04-07 · 类型：研究/模型 · 分类：agent-dev*

**核心贡献：** Claude Opus 4.6 引入 1M token 长上下文（beta），强化 agentic coding、长任务规划和复杂推理能力。

**方法与能力：**
- 更强的多步骤任务规划与工具调用
- 在大型代码库中的调试与代码审查能力显著提升
- 支持 agent teams 与长时间 autonomous workflows

**实验结果：**
- Terminal‑Bench 2.0 agent coding 评测排名第一
- Humanity’s Last Exam 多学科推理测试领先
- GDPval‑AA 知识工作任务评测领先约 144 Elo

**意义：** 强化长上下文与 agent workflow 能力，面向复杂软件工程与知识工作自动化场景。
