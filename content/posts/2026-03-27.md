#### 📌 [Inside our approach to the Model Spec](https://openai.com/index/our-approach-to-the-model-spec)
*发布：2026-03-25 · 类型：经验/实践 · 分类：safety*

**一句话概括：** OpenAI 将 Model Spec 定义为公开的模型行为框架，用于规范不同来源指令冲突时的决策逻辑，并在安全约束下保持用户可控性。

**核心机制：**

- **Chain of Command（指令优先级）**：模型可能同时接收 OpenAI、开发者和用户的指令，通过 authority level 解决冲突。
- **Hard Rules vs Defaults**：少量不可覆盖的安全规则 + 大量可被用户覆盖的默认行为策略。
- **Decision Rubric**：为灰色场景提供决策参考，例如优先选择可逆操作、减少不可预期副作用。

**意义：**

随着模型逐渐具备 agent 能力并执行现实世界操作，AI 行为需要类似“宪法”的公开规范，使外界能够审查模型价值选择并参与改进。

#### Introducing the OpenAI Safety Bug Bounty program
*发布：2026-03-25 · 类型：产品/安全 · 分类：safety*

**发布内容：**

OpenAI 启动 **Safety Bug Bounty** 计划，专门奖励发现 AI 安全与滥用风险的研究者。

**重点覆盖风险：**

- Agent prompt injection 与数据外泄
- Browser / Agent 自动执行危险操作
- MCP 生态中的第三方攻击路径
- 账户信任机制与平台完整性绕过

**关键特点：**

传统 jailbreak 不在奖励范围内，只有 **能够导致真实用户风险的漏洞** 才会被奖励。

**意义：**

表明 AI 安全研究的重点正在从模型输出控制转向 **agent 系统安全与现实世界攻击面**。
